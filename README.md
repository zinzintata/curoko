# 仮想組織 (Virtual Organization)

Claude Code 上で動作する **21部門構成の仮想組織** です。社長(あなた)の指示を受けて、各専門部門(サブエージェント)が自動で業務を遂行します。

## 使い方

1. このフォルダで Claude Code を起動
   ```bash
   cd organization
   claude
   ```
2. 「〇〇して」と普通に指示するだけ。Claude が適切な部門を選出・実行します
3. 部門名を指定する必要はありません(指定してもOK)

## 指示の例

| あなたの一言 | 自動で起動する部門 |
|---|---|
| 「カフェのLP作って」 | Web制作部(ディレクター + プランナー + デザイナー + コピー + FE + SEO + QA) |
| 「AIの最新動向調べて」 | リサーチ部 |
| 「提案書のたたき台」 | 営業部 + リサーチ部(並列) |
| 「新サービスのアイデア出して」 | 新規事業開発部 + リサーチ部 + マーケ部(並列) |
| 「プライバシーポリシー作って」 | 法務部 |
| 「月初の請求書作成」 | 経理部 |

## ファイル構成

- `CLAUDE.md` — Claude向け運営マニュアル (最重要、編集注意)
- `ORG_CHART.md` — 組織図(人間向け詳細版)
- `.claude/agents/*.md` — 各部門のサブエージェント定義
- `departments/<dept>/` — 各部門の成果物保管
- `records/` — 活動記録 (組織の記憶)
- `shared/` — 共通リソース (テンプレート、素材、ナレッジ)

## 新規リポジトリへの移行

この `organization/` フォルダ全体を、新しく作成した GitHub リポジトリの **ルート** に配置すれば、そのまま動作します。

```bash
# 新しいリポジトリ作成後
cp -r organization/* /path/to/new-repo/
cp -r organization/.claude /path/to/new-repo/
cd /path/to/new-repo
claude
```

## カスタマイズ

- **部門の追加**: `.claude/agents/<new-agent>.md` を追加し、`CLAUDE.md` のルーティング表に記載
- **部門の調整**: 既存の `.claude/agents/*.md` の `description` や本文を編集
- **モデル変更**: 各エージェントの YAML frontmatter の `model:` を変更 (opus/sonnet/haiku)

## 制約事項

- セッションをまたいだ自動稼働は不可(ユーザーが起動したセッションでのみ動作)
- 記憶はすべて `records/` ファイルに依存
- 法務・経理の成果物は専門家レビュー前提
